# rentcar00_OPS 현황판 레이어 설계안 v1

## 1. 문서 목적
이 문서는 현황판 레이어 기준 문서를 하나로 묶는다.

포함 범위:
- 현황판 분리 원칙
- 시트 기준
- 기존 AppSheet 5탭 복제 원칙
- `시트1` UI 매핑
- 중요 참고 경로

이 문서는 현황판 레이어만 다룬다.
예약 레이어 세부 설계/상태 전이/DB 규칙은 별도 문서에서 본다.

## 2. 최상위 원칙
- 현황판은 재설계보다 **기존 AppSheet 구조 복제**를 우선한다.
- 현황판은 사장님이 이미 익숙하게 쓰는 운영 화면이므로 탭 이름/순서/정보 배치/진입 흐름을 최대한 유지한다.
- 상단에서는 `예약 / 현황판` 두 레이어만 전환한다.
- 현황판 내부에서는 **기존 하단 5탭 구조를 그대로 유지**한다.
- 보기 좋게 다시 꾸미는 것보다, 기존에 보이던 위치와 정보 밀도 보존을 우선한다.

## 4. 고정 결정
1. 앱 상단에는 `예약 / 현황판` 스위치가 있다.
2. `현황판` 레이어에 들어오면 기존 AppSheet 하단 5탭을 그대로 유지한다.
3. 하단 5탭 이름과 순서는 아래로 고정한다.
   - `대기`
   - `보험`
   - `일반`
   - `장기`
   - `일정`
4. 카드/행에서 보이던 정보 위치와 밀도도 최대한 동일하게 맞춘다.
5. 상세 진입 구조 역시 기존 현황판 흐름을 우선 보존한다.
6. 현황판의 1차 source of truth 는 `차량현황 > 시트1` 이다.
7. `대기 / 보험 / 일반 / 장기` 4탭은 우선 `시트1.상태` 원문값 기준으로 분류한다.
8. `일정` 탭은 `일정` 시트 active row 리스트를 별도 피드로 본다.

## 5. 화면 구조 기준
### 앱 전체
- 상단: `예약 / 현황판` 2버튼 스위치
- `예약` 선택 시: 기존 OPS 예약 처리 구조
- `현황판` 선택 시: 기존 AppSheet 현황판 복제 구조

### 현황판 메인
- 본문: 현재 탭 차량 리스트
- 하단: 기존 AppSheet와 동일한 5탭
- 같은 화면 구조 안에서 탭만 전환되는 형태를 우선한다.

### 리스트/카드 원칙
- 카드보다 리스트성 보드에 가깝게 유지한다.
- 차량번호, 차종, 상태, 대여/반납, 임차인, 위치, 세차 상태 등 현재 보이는 항목을 같은 우선순위로 노출한다.
- 정보 축약은 가능하지만 정보 순서와 의미가 달라지면 안 된다.
- 처음 버전에서는 과한 디자인 재해석을 금지한다.

### 상세 원칙
- 리스트에서 차량 선택 시 상세 진입
- 상세는 차량 상태 / 대여반납 / 임차인 / 위치 / 비고 / 상태액션 흐름을 유지한다.
- 현황판 상세는 예약 레이어 상세와 완전히 같을 필요는 없다.

## 8. 탭별 필터 추정 복원 기준
### 대기
- 1차 필터: `시트1.상태 = '대기'`
- 현재 관찰 특징:
  - `임차인` 공란이 대부분
  - `주차지` 값이 자주 존재
  - `상태액션`은 주로 `반납`
- 화면 구조:
  - 차종별 그룹
  - 컬럼형 리스트
  - 핵심 노출: `차량번호 / 차종 / 세차 / 주차지`

### 보험
- 1차 필터: `시트1.상태 = '보험'`
- 현재 관찰 특징:
  - `배차지` 값이 상대적으로 자주 존재
  - `상태액션`은 `배차 보험` 형태
- 화면 구조:
  - 카드형 리스트
  - 핵심 노출: `업체/주소 성격 정보 + 차량번호 / 차종 / 일자`

### 일반
- 1차 필터: `시트1.상태 = '일반'`
- 현재 관찰 특징:
  - `상태액션`은 `배차 일반` 형태
  - `대여일` 값이 있고 `반납일`은 비거나 미래값
- 화면 구조:
  - 테이블형 리스트
  - 핵심 노출: `차량번호 / 차종 / 대여 / 반납`

### 장기
- 1차 필터: `시트1.상태 = '장기'`
- 현재 관찰 특징:
  - `임차인` / `비고` 값 활용도가 높음
  - `상태액션`은 `배차 장기` 형태가 많음
- 화면 구조:
  - 테이블형 리스트
  - 핵심 노출: `차량번호 / 차종 / 반납년월일 / 임차인`

### 일정
- 1차 피드: `일정` 시트
- 현재 `일정` 시트 실 row 수: `44`
- `Status` 분포:
  - `반납`: `25`
  - `배차`: `16`
  - `기타`: `3`
- 현재 active 후보:
  - `일정완료 != TRUE` 기준 `11`
  - 캡처의 `일정(10)` 과 거의 유사
- 1차 필터 추정:
  - `일정완료 != TRUE`
  - `Status in ('배차', '반납')`
  - 날짜 오름차순 정렬
- 화면 구조:
  - 날짜별 그룹 리스트
  - 핵심 노출: `시간 / 차량번호 / 장소 / 상태`

주의:
- `일정(10)`은 캡처 시점 기준 수치라 현재 실시간 row 수와 1~2건 차이날 수 있다.
- 따라서 `일정`도 숫자 고정보다 **active 일정 조건식**을 복원 기준으로 잡는다.

## 10. 정규화 필요 항목
### 상태
- `상태` 값은 AppSheet 탭 필터와 연결될 가능성이 높다.
- 탭 분류 전 원문 상태를 그대로 보존한다.

### 세차 값
샘플에서 값이 섞여 있다.
- `TRUE`
- `FALSE`
- `N`

권장:
- 내부 표시값은 boolean 성격으로 정규화
- 원문 값도 별도 보존

### 날짜 형식
샘플에서 형식이 섞여 있다.
- `2025-11-14`
- `2026-04-30 0:00:00`
- `11월25일`

권장:
- 원문 보존
- 내부 비교용 parsed date 별도 생성
- parse 실패 시 원문 그대로 표시

### 상태액션
- `상태액션`은 최근 이벤트 로그에 가깝다.
- 메인 리스트에서는 일부만 노출하고,
- 상세에서 전체 텍스트를 보여주는 방향이 안전하다.

## 11. 상세 화면 복원 기준
- 상단 제목은 선택 차량번호 중심
- 상단 액션은 버튼 그리드 구조 유지
- 차량 상세에서는 `Related 일정` 리스트를 상단 블록으로 우선 배치한다
- 일정 탭 row 는 차량 상세가 아니라 일정 상세로 진입한다
- 일정 상세에서는 예약번호를 눌러 예약 상세로 이어지는 흐름을 우선한다
- 상세도 예약 레이어 공용 상세를 그대로 쓰지 말고 현황판 전용 정보 배치를 우선한다

## 12. 현재 구현 반영 상태 (2026-05-11)
- 상단 헤더는 `빵빵카 + 예약/현황판 스위치 + sync` 1줄 구조로 축소됨
- 현황판 탭 상단 설명 문구 제거됨
- 대기 탭은 한줄형 row 구조까지 1차 반영됨
- 차량 상세는 차량등록일 / 차량검사일 / 차령만료일 / 차량번호 세부 필드까지 1차 확장됨
- 일정 탭은 일정 상세 분기와 예약 상세 연결 1차 반영됨
- 차량 상세의 related 일정은 상단으로 이동했고 오늘 이전 일정 제외 1차 반영됨
- 기준 잠금 커밋: `10e2e04`
