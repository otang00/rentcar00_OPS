## 현재 active 작업
**기능 버튼 UI 일관성 정리**

## 목적
예약상세/상태보드에 기능 버튼이 계속 추가될 예정이므로, 상태를 바꾸는 주요 lifecycle 버튼과 보조 기능 버튼의 시각 우선순위를 분리한다.

## 현재 상태 파악
- 예약상세 기능 카드는 2열 그리드 카드형이다.
- 현재 버튼 예:
  - `수정`
  - `차량변경`
  - `배차완료`
  - `반납완료`
  - `전화`
  - `문자`
  - `IMS추가` / `IMS등록됨`
- `수정`, `배차완료`, `반납완료`, `IMS추가`가 모두 강조색 계열이라 중요도가 섞인다.
- 상태보드 즉시배차/세차는 리스트 카드형 다이얼로그를 사용한다.
- 반납완료는 OPS 상태뿐 아니라 IMS active binding까지 건드릴 수 있어 일반 보조 버튼과 같은 레벨로 두면 실수 가능성이 있다.

## 잠긴 UI 기준
### 1. 주요 lifecycle 영역
- 예약상세 상단에 별도 영역으로 둔다.
- 상태에 따라 하나만 표시한다.
  - 미배차: `배차완료`
  - 배차중: `반납완료`
- 한 줄 큰 버튼으로 표시한다.
- 실행 전 확인 다이얼로그를 반드시 띄운다.
- 색상/아이콘은 다른 보조 버튼보다 강하게 두되, `수정`/`IMS추가`와 같은 강조색을 공유하지 않는다.

### 2. 관리 액션 영역
- `수정`, `차량변경`, `IMS추가`/`IMS등록됨`을 묶는다.
- 작은 카드형 버튼으로 유지한다.
- 관리 액션은 상태 변경 lifecycle 버튼보다 시각 우선순위를 낮춘다.

### 3. 연락 액션 영역
- `전화`, `문자`를 묶는다.
- 작은 카드형 버튼 또는 톤다운 버튼으로 둔다.
- lifecycle/관리 액션과 분리해 실수 클릭 가능성을 낮춘다.

### 4. 다이얼로그 일관성
- 즉시배차/세차처럼 선택지가 있는 기능은 리스트 카드형 다이얼로그를 유지한다.
- 완료/취소성 action은 확인 다이얼로그를 사용한다.
- 다이얼로그 문구는 `무엇이 바뀌는지`와 `IMS 동시 처리 여부`를 같이 보여준다.
