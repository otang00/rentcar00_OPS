## 완료된 큰 기능 묶음

### 1. 예약/차량/일정 기본 운영판
- 예약판 5개 탭과 차량 현황판을 운영 기준으로 정리했다.
- 예약 원장, 차량 상태, 배차/반납 일정, 검색/상세 진입 흐름이 연결돼 있다.
- 차량상세에서 즉시배차/상태수정/연관일정/예약생성 흐름을 사용할 수 있다.

### 2. 관리자/직원/차량관리
- 관리자 `rentcar00` 기준의 직원관리 MVP를 구현했다.
- 직원 활성상태, 권한, 위치/활동 정보, 관리자 표시용 비밀번호 확인 UI를 만들었다.
- 관리자 차량관리에서 차량 추가/수정/삭제, 기본/고급 컬럼 관리를 할 수 있다.
- 차량 추가 기준은 차량번호 필수, 초기 상태 `대기중`, 세차/실내세차 `FALSE`다.

### 3. IMS 연동
- 일반예약 IMS 가져오기: 차량번호/날짜 기준 조회, 카드형 후보 선택, OPS 예약 생성 흐름을 구현했다.
- 파트너콜 형태 예약은 `GET /v2/rent-requests/{detailId}`까지 조회해 고객명/번호/생년월일/소개처/가격/배차지/반납지를 채운다.
- 보험배차 IMS 가져오기: `GET /v2/rencar-claims` + `periodOption=using_car` 기준으로 구현했다.
- IMS 생성/차량변경/삭제 관련 중간서버 API와 OPS 연동 기준을 잡았다.
- IMS 계정 누락 문제는 parser `.env` 복구 및 재시작으로 해결했다.

### 4. 홈페이지 예약 이벤트 수신
- endpoint: `POST /api/integrations/rentcar00/reservation-events`
- HMAC 서명, timestamp, eventId, payload schema 검증을 넣었다.
- 정상 payload는 `rc00_ops_reservation_events`에 저장한 뒤 예약 원장/상태/배차·반납 일정까지 자동 생성한다.
- 앱은 `homepage_review=pending` 배지와 상단 `홈페이지 N` 표시를 보여준다.
- 예약 상세에서 `홈페이지확인` 처리 가능하다.

### 5. UI/배포 정리
- 앱 아이콘은 `빵빵카` 단독 표기로 정리했다.
- 차량상세 연관일정은 시간순 세로 카드로 정리했다.
- 예약생성 다이얼로그 하단 버튼 줄바꿈 문제를 개선했다.
- 상단 검색 버튼으로 일정/차량/예약 통합 검색 화면에 바로 진입할 수 있다.
- 검색 결과는 일정 → 차량 → 예약 순으로 나뉘어 표시된다.
- 일정탭 예약 연결 일정은 배차 파랑 / 반납 빨강 강조 카드로 더 강하게 보인다.
- GDrive APK는 최신 1개만 유지한다.

### 6. 시간 처리 KST 통일
- 예약/일정/현황판 구조는 유지했다.
- 앱 입력/표시는 KST 벽시계 시간으로 통일했다.
- DB 저장 시에만 KST → UTC timestamp 변환한다.
- DB 조회/표시는 UTC → KST로 복원한다.
- 공통 helper: `lib/shared/utils/ops_kst_datetime.dart`
- 검증: `flutter analyze`, `flutter test`, `flutter build apk --release --target-platform android-arm64` 통과.



### 9. 수동 새로고침 + 핵심 Realtime 1차
- Realtime은 로그인 세션 생성 후 구독하고 실패 시 재구독한다.
- 새로고침 가능한 화면은 `anywhere` 당겨서 새로고침 기준이다.
- 예약 탭, 현황판 탭, 검색, 예약상세, 차량/일정 상세에 당겨서 새로고침을 추가했다.
- 앱 전역 `OpsRealtimeRefreshBridge`에서 핵심 테이블 변경을 구독한다.
- 핵심 구독 대상은 `rc00_ops_reservations`, `rc00_ops_reservation_states`, `rc00_ops_schedules`, `rc00_ops_cars`다.
- 로그 테이블 realtime은 메인 화면 과다 갱신 방지를 위해 이번 phase에서 제외했다.
- 앱 복귀 시 핵심 provider를 1회 갱신한다.
- 예약상세/차량상세/일정상세의 주소값은 네이버지도 검색 링크로 열 수 있다.
- 일정 생성/수정에서 차량선택과 선택안함을 지원한다.

### 8. 직원 액션 로그 1차
- DB `rc00_ops_action_logs`를 예약 전용에서 직원 감사 로그용으로 확장했다.
- 예약/차량/일정/직원관리 주요 변경 액션은 현재 로그인 직원 기준으로 기록한다.
- 관리자 > 작업로그에서 최신 작업 이력을 확인한다.
- 예약 상세에서도 해당 예약의 업무 로그를 확인한다.

### 7. 운영 DB 정리
- 차량번호 오기 `141호4780 → 142호4780`을 차량/예약/일정 연결 기준으로 정리했다.
- 신규 차량 2대 추가 완료:
  - `165허8095` / 모닝 / 상태 `대기중`
  - `175호2135` / 스타리아 / 상태 `대기중`
- 등록증 기반 상세값은 각 차량 `payload_json`에 보관했다.

- 예약판 배차대기 시간이 한국시간 기준으로 표시되는지 확인.
- 일반예약 생성/수정/취소 흐름 확인.

## 현재 리스크 / 주의점
1. 운영 secret과 runtime 설정은 승인 없이 수정하지 않는다.
2. 홈페이지 차량 추가/동기화는 OPS repo에는 아직 없다. 홈페이지 쪽 별도 구조 확인 필요.
3. IMS 차량 등록은 복잡한 외부 상태 변경이므로 수동 처리 우선.
4. 차량 삭제는 과거 예약/일정 연결 리스크가 있어 운영 전 재확인한다.
5. b48 실기기 확인 전까지는 배포물 준비/업로드 완료 상태로 본다.
