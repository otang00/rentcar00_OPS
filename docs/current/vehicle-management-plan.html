차량관리 기능 구현 계획

관리자 홈에서 차량 추가, 삭제, 전체 컬럼 조회/수정이 가능하도록 만드는 1차 구현 계획입니다. 승인 전에는 구현하지 않습니다.

목적

관리자 홈 > 차량관리에서 차량 추가 / 삭제 / 모든 컬럼 조회·수정을 가능하게 합니다.

대상 테이블: rc00_ops_cars

접근 가능 컬럼 기준

car_number
car_name
status
status_action
car_wash
interior_wash
start_at / end_at
start_at_ts / end_at_ts
customer_name
customer_phone
pickup_location
parking_location
note_text
car_registered_at
car_inspection_at
car_age_expiry_at
car_number_front
car_number_middle
car_number_rear
payload_json

실제 현재 스키마 기준 source/import 계열 컬럼은 제거된 상태입니다. 생성/수정일 등 시스템 성격 컬럼은 고급 영역과 경고를 붙입니다.

Phase 1. 실제 DB 스키마 확정

  • rc00_ops_cars 실제 컬럼, nullable, default, RLS 확인
  • 앱에서 일반 영역 / 고급 영역으로 나눌 컬럼 분류
  • 삭제 시 예약/일정 연결 영향 확인

종료 조건 앱 노출 컬럼과 위험 컬럼 분류 완료

Phase 2. 차량관리 목록 화면

  • 관리자 홈 차량관리 카드에 route 연결
  • 차량 목록 카드형 표시
  • 검색: 차량번호, 차종, 상태, 주차위치
  • 관리자만 접근 가능

종료 조건 관리자 홈에서 차량관리 목록 진입 가능

Phase 3. 차량 추가

  • 새 차량 row 생성
  • 필수값: 차량번호
  • 차량번호 중복 방지
  • 나머지 컬럼은 선택 입력

종료 조건 추가 후 목록 자동 갱신

Phase 4. 차량 수정

  • 운영 컬럼 전체 수정 가능
  • 날짜/상태/세차/위치/고객/메모 수정
  • payload_json은 JSON 입력 검증 후 저장
  • source/import 계열은 고급 영역에서 수정

종료 조건 저장 후 DB 반영 및 목록 갱신

Phase 5. 차량 삭제

  • 관리자만 삭제 가능
  • 삭제 전 확인 다이얼로그
  • 예약/일정 연결 가능성 경고 표시
  • 요청 범위상 실제 delete 버튼 제공

종료 조건 삭제 후 목록에서 제거

Phase 6. 검증 / 문서 / 커밋

  • flutter analyze
  • flutter test
  • current/completed 문서 업데이트
  • 검증 통과 후 커밋

주의 리스크

  • 차량 삭제는 과거 예약/일정 표시와 연결이 끊길 수 있습니다.
  • 생성일/수정일/payload_json 같은 시스템 성격 컬럼 수정은 추적성을 깨뜨릴 수 있습니다.
  • 그래서 모든 컬럼 접근은 제공하되, 위험 컬럼은 고급 영역과 경고로 분리합니다.
