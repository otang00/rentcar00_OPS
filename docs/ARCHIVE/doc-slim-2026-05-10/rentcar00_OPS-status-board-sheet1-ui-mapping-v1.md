## 5. 행 UI 권장 구조
### 기본 행
1. 좌측 상단: `차량번호`
2. 그 옆 또는 아래: `차종`
3. 우측 강조: `상태`
4. 본문 1줄: `임차인`
5. 본문 2줄: `대여일 ~ 반납일`
6. 본문 3줄: `배차지 / 주차지`
7. 보조 배지: `세차`, `실내세차`
8. 필요 시 하단 메모: `비고` 일부

- 그래서 큰 카드보다는 **얇은 리스트 행**을 우선한다.

## 7. 정규화 필요 항목
### 7-1. 상태
- `상태` 값은 AppSheet 탭 필터와 연결될 가능성이 높다.
- 현황판 탭 분류 전 원문 상태를 그대로 보존한다.

### 7-2. 세차 값
현재 샘플에서 값이 섞여 있다.
- `TRUE`
- `FALSE`
- `N`

권장:
- 내부 표시값은 boolean 성격으로 정규화
- 원문 값도 별도 보존

### 7-3. 날짜 형식
현재 샘플에서 형식이 섞여 있다.
- `2025-11-14`
- `2026-04-30 0:00:00`
- `11월25일`

권장:
- 원문 보존
- 내부 비교용 parsed date 별도 생성
- parse 실패 시 원문 그대로 표시

### 7-4. 상태액션
- `상태액션` 은 단순 표시 필드가 아니라 사실상 최근 이벤트 로그에 가깝다.
- 메인 리스트에서는 일부만 노출하고,
- 상세에서 전체 텍스트를 보여주는 방향이 안전하다.

## 8. 현황판과 예약 레이어 관계
- 현황판 메인 분류 기준은 `시트1`
- 예약 처리 기준은 `예약`
- 일정 이벤트 기준은 `일정`

즉,
- 현황판 = 차량 중심 보드
- 예약 = 예약 중심 처리
- 일정 = 이벤트 보조
