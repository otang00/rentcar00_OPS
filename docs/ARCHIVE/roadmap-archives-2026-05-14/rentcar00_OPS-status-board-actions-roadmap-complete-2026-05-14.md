# rentcar00_OPS Status Board Actions Roadmap

## 1. 문서 역할
이 문서는 현황판 차량 상태별 액션 분기와 반납 상태복귀 구현 기준을 잠그는 문서다.

## 2. 목표
- 대기 차량에는 배차 관련 기능만 보이게 한다.
- 보험/일반/장기 운행 차량에는 반납 기능이 보이게 한다.
- 반납 시 차량 상태를 대기 기준으로 안전하게 복귀시킨다.

## 3. 현재 코드 상태
### 3-1. 이미 구현된 것
- 차량 상세에서 예약생성 가능
- 보험/일반/장기 즉시 상태전환 dialog 가능
- 세차 토글 가능
- 주차지 수정 가능

### 3-2. 아직 없는 것
- 차량 상태에 따른 기능 노출 분기
- 운행 차량 전용 `반납 완료` 액션
- 반납 시 세차상태 초기화
- 반납 시 주차지 기본값 복귀

## 4. 잠글 규칙
### 대기 차량
노출:
- 예약생성
- 보험대차
- 일반대차
- 장기대차
- 세차 관련
- 주차지 수정

### 보험 / 일반 / 장기 차량
노출:
- 반납 완료
- 세차 관련
- 주차지 수정

## 5. 반납 완료 규칙
반납 완료 실행 시:
- `status = 대기중`
- `status_action = 반납 완료`
- `car_wash = FALSE`
- `interior_wash = FALSE`
- `parking_location = 수푸레`

## 6. 구현 포인트
### UI
- `status_board_detail_page.dart`
- 상태/탭 기준으로 액션 그룹 분기

### 저장 로직
- `supabase_ops_repository.dart`
- 반납 완료 전용 메서드 추가 권장

## 7. 리스크
- 반납 시 고객/운행 정보까지 비울지 범위 확인 필요
- `tab` 기준과 `status` 기준이 어긋나는 데이터가 있을 수 있음

## 8. 종료 조건
- 대기 차량과 운행 차량에서 버튼 구성이 다르게 보임
- 반납 완료 후 상태/세차/주차지가 기대값으로 바뀜
- 현황판 refresh 후 대기 상태로 확인 가능
