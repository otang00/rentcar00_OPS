# rentcar00_OPS 앱 설계 준비

## 목적
- 사장님이 줄 설계 요구사항을 구조적으로 받기 위한 기준 문서
- Flutter Android 운영앱의 화면/권한/데이터/API/상태 흐름 설계 준비

## 현재 기준점
- 프로젝트 경로: `projects/rentcar00_OPS`
- 앱 내부 프로젝트명: `rentcar00_ops`
- Android 개발 환경: 1차 구성 완료
- Flutter analyze / debug APK build 검증 완료

## 이번 설계에서 먼저 잠글 것
1. 앱의 핵심 목적
2. 사용자 유형
3. 핵심 화면 목록
4. 각 화면에서 하는 일
5. 서버/API 연동 여부
6. 로그인/권한 필요 여부
7. 오프라인 필요 여부
8. 알림/푸시 필요 여부
9. 파일 업로드/카메라 사용 여부
10. 초기 출시 범위(MVP)

## 설계 입력 템플릿
### 1. 앱 목적
- 이 앱은 무엇을 가장 먼저 해결해야 하는가?

### 2. 사용자
- 누가 쓰는가?
- 관리자 전용인가?
- 직원별 권한 차이가 있는가?

### 3. 핵심 기능
- 반드시 있어야 하는 기능
- 있으면 좋은 기능
- 지금은 빼야 하는 기능

### 4. 화면 목록
- 로그인
- 홈/대시보드
- 예약 목록
- 예약 상세
- 차량 목록
- 차량 상세
- 배차/반납 처리
- 정비/상태 변경
- 고객 검색
- 설정
- 기타 필요한 화면

### 5. 데이터/API
- 기존 서버가 있는가?
- 기존 웹/ERP와 같은 DB를 쓰는가?
- 새 API를 만들 것인가?
- 필요한 주요 데이터는 무엇인가?

### 6. 운영 정책
- 실시간 반영이 필요한가?
- 오프라인 저장이 필요한가?
- 사진 첨부가 필요한가?
- 직원 행동 로그가 필요한가?

### 7. MVP 범위
- 1차 출시에서 꼭 들어갈 것
- 2차로 미뤄도 되는 것

## 설계 산출물 예정
- IA(화면 구조도)
- 기능 우선순위 표
- 데이터 엔티티 초안
- API 연동 초안
- Flutter 폴더 구조 초안
- 상태관리/라우팅 전략
- MVP 개발 phase
