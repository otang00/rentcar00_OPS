# rentcar00_OPS docs

## 읽는 순서
1. `docs/current/rentcar00_OPS-current.md`
   - 지금 실제로 실행 중인 작업 1건
2. `docs/current/rentcar00_OPS-main.md`
   - 프로젝트 전체 구조 / 정책 / spec / 운영 기준
3. `docs/completed/rentcar00_OPS-completed.md`
   - 완료된 기능의 운영/검증/장애 대응 요약
4. 필요할 때만 `docs/past/`

## 잠금 규칙
- current: 실행 문서 1개만 유지
- main: 프로젝트 메인 기준 문서 1개만 유지
- completed: 완료 기능 누적 문서 1개만 유지
- past: 과거 설계/아이디어/스냅샷만 보관
