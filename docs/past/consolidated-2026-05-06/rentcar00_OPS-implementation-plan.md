# rentcar00_OPS Implementation Plan

## 1. 목적
이 문서는 실제 구현 준비를 위한 최소 작업 순서를 정리한다.
문서 확장보다, 바로 만들 수 있는 기준을 잠그는 데 목적이 있다.

---

## 2. 구현 전 확정 순서
이 순서는 바꾸지 않는다.

1. 탭별 기능 리스트 확정
2. 탭별 카드 표시 항목 확정
3. 탭별 버튼/상태 전이 확정
4. 시트 상태 체크 반영 항목 확정
5. 그 다음에만 DB/스키마 확정

---

## 3. 구현 대상 순서
### Phase 1. 탭 기준 고정
대상:
- 예약중
- 오늘배차
- 배차중
- 반납일
- 완료

이 단계에서 확정할 것:
- 탭 진입 조건
- 카드에 보여줄 정보
- 카드에서 바로 누를 버튼
- 버튼 후 이동 상태

종료 조건:
- 5개 탭 모두 기능 리스트가 확정됨

### Phase 2. 카드 액션 고정
이 단계에서 확정할 것:
- 기능별 버튼 위치
- 버튼 누르면 바뀌는 업무 상태
- 로그 저장 여부
- 시트 반영 여부

종료 조건:
- 기능별 액션 표가 확정됨

### Phase 3. 보조 화면 고정
이 단계에서 확정할 것:
- 상세 화면에서만 처리할 항목
- 문자/요청문 생성 흐름
- 완료 기록 조회 방식

종료 조건:
- 보조 화면 역할이 메인 흐름과 분리됨

### Phase 4. 데이터 구현 준비
이 단계에서 확정할 것:
- 시트에서 읽을 값
- Supabase에 저장할 값
- 최소 상태 체크 반영 규칙

종료 조건:
- 구현 가능한 read/write 기준이 정리됨

---

## 4. 바로 다음 작업
다음 문서는 아래 순서로 작성한다.
1. 탭별 기능 리스트 v1
2. 탭별 카드 항목 v1
3. 탭별 버튼/상태 전이 v1

이 3개가 끝나기 전에는 DB 세부 설계를 확정하지 않는다.
