# rentcar00_OPS 현황판 복제 기준 v1

## 1. 문서 목적
이 문서는 기존 AppSheet `현황판` 화면을 Flutter 앱 안으로 옮길 때, 화면 구조와 운용 흐름을 최대한 동일하게 유지하기 위한 복제 기준 문서다.

## 2. 최상위 원칙
- 현황판은 재설계보다 **기존 AppSheet 구조 복제**를 우선한다.
- 현황판은 사장님이 이미 익숙하게 쓰는 운영 화면이므로, 탭 이름/순서/정보 배치/진입 흐름을 최대한 유지한다.
- 상단에서는 `예약 / 현황판` 두 레이어만 전환하고,
  현황판 내부에서는 **기존 하단 5탭 구조를 그대로 유지**한다.
- 현황판은 보기 좋게 다시 꾸미는 것보다, **기존에 보이던 위치와 정보 밀도 보존**을 우선한다.

## 3. 데이터 기준
- spreadsheet: `차량현황`
- 현황판 기준 탭: `시트1`
- 보조 탭:
  - `예약`
  - `일정`

현재 확인된 시트 목록:
- `시트1`
- `예약`
- `일정`

중요:
- 현황판 레이어의 1차 source of truth 는 `시트1` 이다.
- `예약` / `일정` 은 현황판 상세 보강이나 연결 판단에 쓰더라도, 메인 보드 분류 기준 자체를 먼저 대체하지 않는다.

## 4. 고정 결정
1. 앱 상단에는 `예약 / 현황판` 스위치가 있다.
2. `현황판` 레이어에 들어오면 기존 AppSheet 하단 5탭을 그대로 유지한다.
3. 하단 5탭의 이름, 순서, 성격은 AppSheet 기준을 우선 복제한다.
4. 카드/행에서 보이던 정보 위치와 밀도도 최대한 동일하게 맞춘다.
5. 상세 진입 구조 역시 기존 현황판 흐름을 우선 보존한다.
6. 현황판은 새 업무 흐름을 발명하지 않고, 기존 운영 습관을 앱 안으로 옮기는 것이 1차 목표다.

## 5. 화면 구조 기준
### 5-1. 앱 전체
- 상단: `예약 / 현황판` 2버튼 스위치
- `예약` 선택 시: 기존 OPS 예약 처리 구조
- `현황판` 선택 시: 기존 AppSheet 현황판 복제 구조

### 5-2. 현황판 메인
- 본문: 현재 탭 차량 리스트
- 하단: 기존 AppSheet와 동일한 5탭
- 같은 화면 구조 안에서 탭만 전환되는 형태를 우선한다.

### 5-3. 리스트/카드 원칙
- 카드보다 리스트성 보드에 가깝게 유지한다.
- 차량번호, 차종, 상태, 대여/반납, 임차인, 위치, 세차 상태 등 현재 보이는 항목을 최대한 같은 우선순위로 노출한다.
- 정보 축약은 가능하지만, 정보 순서와 의미가 달라지면 안 된다.
- 처음 버전에서는 과한 디자인 재해석을 금지한다.

### 5-4. 상세 원칙
- 리스트에서 차량 선택 시 상세 진입
- 상세는 차량 상태 / 대여반납 / 임차인 / 위치 / 비고 / 상태액션 흐름을 유지한다.
- 현황판 상세는 예약 레이어 상세와 완전히 같을 필요는 없다.

## 6. 구현 우선순위
### Phase 1
- 하단 5탭 이름/순서 고정
- 탭별 기본 리스트 복제
- `시트1` 컬럼을 그대로 읽어 행 UI 구성

### Phase 2
- 탭별 필터 규칙 정리
- 상세 화면 구조 고정
- `상태액션` / `비고` / 위치 정보 보강

### Phase 3
- `예약` / `일정` 과 연결해 상세 보강
- 검색, 강조, 배지 등 운영 편의 기능 보강

## 7. 현재 확인된 리스크
- 캡처만으로는 하단 5탭 각각의 정확한 필터 조건이 아직 잠기지 않았다.
- `시트1` 의 `상태` 값과 AppSheet 탭 조건이 1:1인지 추가 확인이 필요하다.
- 기존 AppSheet에서 virtual column / slice / action 으로 보정하는 규칙이 있을 수 있다.
- 따라서 현황판 1차 문서는 **구조 복제 우선**, 조건식 완전 복원은 후속 작업으로 분리한다.

## 8. 다음 확인 필요 항목
- 기존 하단 5탭의 정확한 이름
- 각 탭의 필터 조건
- 각 탭에서 강조되는 필드 차이
- 탭별 정렬 기준
- 리스트 터치 후 상세/액션 흐름
