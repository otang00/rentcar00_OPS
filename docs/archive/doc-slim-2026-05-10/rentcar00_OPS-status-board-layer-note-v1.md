# rentcar00_OPS 현황판 레이어 추가 메모 v1

## 1. 문서 목적
이 문서는 `rentcar00_OPS`의 **현황판 레이어 전용 방향 메모**다.

원칙:
- 이 문서는 현황판 레이어만 다룬다.
- 예약 레이어 세부 설계/상태 전이/DB 규칙은 여기서 다루지 않는다.

## 2. 배경
- 기존에는 AppSheet 현황판과 Flutter OPS 앱이 분리되어 있었다.
- 예약 처리 구조는 이미 상당 부분 정리되었지만, 실제 연결 단계에서 기존 현황 시트 구조가 복잡하게 얽혀 유지보수 부담이 커졌다.
- 따라서 AppSheet를 계속 병행 유지하기보다, 하나의 앱 안에서 `예약`과 `현황판`을 함께 운영하는 통합 구조를 검토한다.

## 3. 현재 방향
- 앱 최상단 영역에서 `예약` / `현황판` 두 버튼으로 레이어를 전환한다.
- 두 버튼은 별도 앱 이동이 아니라, **같은 앱 안의 두 운용 레이어를 스위치하는 진입점**으로 본다.
- `예약` 레이어는 기존 OPS 앱의 예약 상태별 처리 흐름을 유지한다.
- `현황판` 레이어는 기존 AppSheet 현황판을 **재설계보다 먼저 최대한 그대로 복제**하는 방향으로 간다.

## 4. 원칙
- `예약`과 `현황판`은 같은 앱 안에 공존하되, 기준 원장과 화면 구조는 분리해서 본다.
- 현황판의 1차 source of truth 는 Google Sheets `차량현황 > 시트1` 이다.
- 현황판은 새 구조 제안보다 **기존 AppSheet 하단 5탭과 보이는 화면 복제**를 우선한다.
- 초기 현황판 구현에서는 디자인 재해석보다 기존 정보 위치와 운영 습관 보존을 우선한다.

## 5. 이번 메모에서 잠그는 최소 결정
1. `rentcar00_OPS`는 단일 앱 안에 `예약` 레이어와 `현황판` 레이어를 함께 가진다.
2. 앱 최상단에서 두 버튼으로 레이어 전환이 가능해야 한다.
3. 현황판 레이어는 기존 AppSheet 하단 5탭을 그대로 유지해야 한다.
4. 현황판 메인 분류 기준은 `차량현황 > 시트1` 이다.
5. 세부 탭 조건과 화면 복제 기준은 후속 문서에서 확정한다.

## 6. 후속 정리 필요 항목
- 기존 하단 5탭의 정확한 이름과 조건
- 탭별 리스트/상세 노출 차이
- `시트1` 컬럼과 AppSheet 화면 요소 매핑
- 예약 레이어와의 연결 지점
