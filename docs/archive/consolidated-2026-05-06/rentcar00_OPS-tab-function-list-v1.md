# rentcar00_OPS 탭별 기능 리스트 v1

## 1. 목적
이 문서는 앱 메인 구조인 **예약 상태별 탭 리스트** 기준으로,
각 탭에서 실제로 어떤 업무를 처리해야 하는지 정리한다.

원칙:
- 메인은 탭 리스트다.
- 상세는 보조다.
- 기능은 "어디서 어떻게 쓰는지" 기준으로 적는다.
- 아직 확정되지 않은 것은 억지로 만들지 않는다.

---

## 2. 탭 구성
- 예약중
- 오늘배차
- 배차중
- 반납일
- 완료

---

## 3. 예약중 탭
### 목적
배차 전 예약 건에서 사전 준비 업무를 처리한다.

### 이 탭에서 바로 해야 하는 기능
- 연락처 저장
- 신분증/주소 요청
- 신분증 확인
- 주소 확인
- 배차준비완료

### 기능 의미
#### 연락처 저장
- 고객 연락처를 휴대폰 연락처에 저장한다.

#### 신분증/주소 요청
- 신분증과 주소 요청을 한 번에 문자 발송하는 액션이다.
- 요청은 하나로 묶고, 이후 확인은 분리한다.

#### 신분증 확인
- 고객이 신분증을 보냈는지 확인 처리한다.

#### 주소 확인
- 고객이 주소를 보냈는지 확인 처리한다.

#### 배차준비완료
- 예약 건의 사전 준비가 끝났음을 표시한다.
- 다음 단계 탭으로 넘기는 기준 액션이다.

### 처리 결과
- 이 탭에서는 주로 **사전 준비 상태**가 바뀐다.
- 준비가 끝나면 다음 단계 탭으로 이동한다.

### 확인 필요
- 예약중에서 오늘배차로 넘어가는 정확한 조건
- 배차준비완료의 완료 기준

## 4. 오늘배차 탭
### 목적
오늘 실제 배차가 필요한 예약을 당일 처리한다.

### 이 탭에서 바로 해야 하는 기능
1. 고객에게 전화
2. 고객에게 배차안내 문자
3. 탁송접수
4. 탁송기사 연락
5. 배차계약서 작성
6. 고객에게 차량확인 및 서명 문자안내

### 기능 의미
#### 고객에게 전화
- 당일 배차 관련 직접 연락을 한다.

#### 고객에게 배차안내 문자
- 고객에게 배차 관련 안내 문자를 발송한다.

#### 탁송접수
- 현재는 탁송접수 문자를 생성하고 카카오톡을 여는 방식으로 처리한다.

#### 탁송기사 연락
- 현재는 추후 확장 대상이다.
- 나중에 탁송 API가 생기면 추가 가능하다.

#### 배차계약서 작성
- IMS API 확인 또는 IMS에서 직접 계약서를 작성하는 액션이다.

#### 고객에게 차량확인 및 서명 문자안내
- 고객에게 차량 확인 및 서명 안내 문자를 발송한다.

### 처리 원칙
- 오늘배차 액션들은 출차 당일 실행 액션들이다.
- 배차계약서 작성과 차량확인·서명 안내는 **필수 순서로 묶지 않고 독립 액션**으로 둔다.

### 처리 결과
- 오늘배차 탭에서는 **당일 출발 실행 상태**가 바뀐다.
- 정상 처리 후 배차중으로 넘어갈 가능성이 높다.

### 확인 필요
- 오늘배차 판정 기준
- 실제 배차중 이동 조건
- 탁송기사 연락의 초기 버전 필요 여부

## 5. 배차중 탭
### 목적
차량이 이용 중인 예약을 관리한다.

### 이 탭에서 바로 해야 하는 기능
1. 고객에게 전화
2. 서명확인
3. 반납 전 안내 문자
4. 반납 일정 변경
5. 고객에게 긴급출동 안내
6. 사고접수

### 기능 의미
#### 고객에게 전화
- 운행 중 확인이나 이슈 대응을 위해 고객에게 직접 연락한다.

#### 서명확인
- 차량 인도 후 고객 서명 완료 여부를 확인하는 액션이다.
- 배차중 운영 확인 항목으로 본다.

#### 반납 전 안내 문자
- 반납일 전 고객에게 반납 관련 안내 문자를 발송한다.

#### 반납 일정 변경
- 반납 일정이 바뀌는 경우 일정 변경 처리를 한다.

#### 고객에게 긴급출동 안내
- 긴급상황 발생 시 고객에게 긴급출동 안내 문자를 전송한다.

#### 사고접수
- 사고 발생 시 렌터카공제조합 전화로 접수하는 액션이다.

### 처리 결과
- 이용중 관리 상태가 바뀐다.
- 반납일이 가까워지면 반납일 탭으로 넘어가는 흐름이다.

### 확인 필요
- 배차중 유지 기준
- 반납 일정 변경 시 원본 일정/상태와 어떻게 맞출지
- 사고접수 후 내부 기록을 남길지 여부

## 6. 반납일 탭
### 목적
반납 예정 또는 반납 처리 직전/직후 업무를 관리한다.

### 이 탭에서 바로 해야 하는 기능
1. 반납안내 문자
2. 탁송접수
3. 반납지 변경
4. 연장시 추가요금 안내
5. 반납완료

### 기능 의미
#### 반납안내 문자
- 고객 확인 후 반납 관련 안내 문자를 발송한다.

#### 탁송접수
- 현재는 탁송 문자를 작성하는 방식으로 처리한다.
- 추후 탁송 API로 연결할 수 있다.

#### 반납지 변경
- 예약 테이블의 반납지를 변경한다.
- 변경된 반납지는 탁송접수 시 반영된다.

#### 연장시 추가요금 안내
- 추가요금을 계산해서 고객에게 안내 문자를 보낸다.

#### 반납완료
- 실제 반납 종료 처리다.
- 완료 탭 이동의 핵심 기준이다.

### 처리 결과
- 반납 완료 시 완료 탭으로 이동한다.
- 최종적으로 시트 상태 체크 반영 대상이 될 가능성이 높다.

### 확인 필요
- 반납일 탭 진입 기준
- 추가요금 계산 기준
- 반납완료 시 시트에 어떤 체크를 반영할지

## 7. 완료 탭
### 목적
반납 완료 후 짧은 유예기간 동안 종료 예약을 확인한다.

### 이 탭에서 봐야 하는 것
- 완료된 예약 기본 정보
- 처리 이력
- 다시 확인할 메모/로그

### 이 탭에서 바로 해야 하는 기능
- 직접 액션은 두지 않는다.
- **조회/검색 중심**으로 본다.

### 유지 기준
- 반납완료 후 **7일간** 완료 탭에 노출한다.
- 7일이 지나면 완료 탭에서는 제외한다.
- 이후에는 검색으로만 조회한다.

### 처리 결과
- 이 탭은 업무 실행보다는 반납 직후 사후확인 성격이 강하다.

### 확인 필요
- 7일 경과 기준 시각을 반납완료 시점으로 볼지 날짜 기준으로 볼지
- 검색 기준

## 8. UI / 공용 상세 기준
### 탭 메인 리스트
원칙:
- 각 탭 메인은 **예약 리스트**다.
- 카드는 두껍지 않게, **한 줄 표시**를 기본으로 한다.
- 리스트 정렬은 **배반차 날짜 기준 내림차순**이다.

리스트 표시 항목:
- 배반차 날짜 / 차량번호 / 배차주소(간략)

정렬 기준:
- 예약중 / 오늘배차 = 배차일 기준
- 배차중 / 반납일 / 완료 = 반납일 기준

메모:
- 고객이름, 전화번호 등 상세 정보는 리스트가 아니라 공용 상세에서 본다.
- 리스트 상태색은 넣지 않는다. 각 탭에는 같은 상태의 예약만 뜬다.

### 공용 예약 상세
원칙:
- 각 탭에서 리스트를 누르면 **공용 예약 상세 화면**으로 들어간다.
- 상세 화면은 모든 탭에서 공용으로 사용한다.
- 예약 상세 정보는 전부 보여준다.
- 상태는 색깔로 표시한다.

레이아웃:
- 상세 상단 = 액션 버튼 영역
- 상세 하단 = 탭 네비게이션
- 하단 탭 구조는 AppSheet 스타일을 기준으로 본다.

### 액션 버튼 노출 규칙
- 기본은 **현재 탭에서 필요한 버튼만 노출**한다.
- 대신 별도로 **전체 액션 보기** 버튼을 둔다.
- 전체 액션 보기에서는 다른 단계 액션도 확인 가능하게 한다.
- 운영 확인 항목은 **체크리스트가 아니라 버튼형 액션**으로 처리한다.
- 버튼을 누른 뒤에는 완료 상태 색으로 바뀌며, 그 자체가 확인 표시 역할을 한다.

버튼 상태 예시:
- 활성 = 초록
- 완료 = 파랑
- 발송 후 미답변 = 노랑
- 비활성 = 회색

### 단계 이동 원칙
- 각 단계 액션을 완료하면 다음 단계 탭에서 볼 수 있어야 한다.

### 문자/요청문 생성
역할:
- 신분증/주소 요청
- 배차 안내
- 탁송 요청
- 기사 안내
- 반납 안내
등의 문구 생성/복사/전달 보조

확인 필요:
- 각 기능이 문자/공유/복사 중 어떤 방식으로 실행되는지

## 9. 지금 문서에서 확정한 것
- 메인은 탭별 상태 리스트다.
- 탭은 예약중 / 오늘배차 / 배차중 / 반납일 / 완료다.
- 각 탭 메인은 예약 리스트다.
- 리스트는 한 줄 표시를 기본으로 한다.
- 리스트 표시값은 배반차 날짜 / 차량번호 / 배차주소(간략)이다.
- 리스트 정렬은 배반차 날짜 기준 내림차순이다.
- 예약중 / 오늘배차는 배차일 기준으로, 배차중 / 반납일 / 완료는 반납일 기준으로 본다.
- 리스트 상태색은 넣지 않는다.
- 각 탭에서 누르면 공용 예약 상세로 들어간다.
- 상세 상단은 액션 버튼, 하단은 탭 네비게이션이다.
- 상세 액션은 현재 탭에 필요한 버튼만 노출한다.
- 대신 전체 액션 보기 버튼을 별도로 둔다.
- 액션 완료 시 다음 단계 탭으로 이동하는 구조로 본다.
- 예약중 탭 액션은 연락처 저장 / 신분증·주소 요청 / 신분증 확인 / 주소 확인 / 배차준비완료로 확정했다.
- 오늘배차 탭 액션은 고객에게 전화 / 고객에게 배차안내 문자 / 탁송접수 / 탁송기사 연락(추후 확장) / 배차계약서 작성 / 고객에게 차량확인 및 서명 문자안내로 확정했다.
- 오늘배차의 배차계약서 작성과 차량확인·서명 안내는 독립 액션으로 본다.
- 배차중 탭 액션은 고객에게 전화 / 서명확인 / 반납 전 안내 문자 / 반납 일정 변경 / 고객에게 긴급출동 안내 / 사고접수로 확정했다.
- 오늘배차에서 배차중은 탁송 출발 시점 기준으로 전이하기로 확정했다.
- 배차중에서 반납일은 반납일 당일 00:00 기준으로 전이하고, 반납 하루 전에는 노란색으로 임박 상태를 표시하기로 확정했다.
- 반납일 탭 액션은 반납안내 문자 / 탁송접수 / 반납지 변경 / 연장시 추가요금 안내 / 반납완료로 확정했다.
- 완료 탭은 직접 액션 없이 조회 중심으로 두고, 반납완료 후 7일간만 노출하기로 확정했다.
- 예약중에서 오늘배차는 배차일이 오늘이면 자동 진입하며, 준비 미완료 건은 주황색으로 구분하기로 확정했다.
- 각 탭은 운영 확인 항목을 전제로 구현하며, 예약중/오늘배차/배차중/반납일 확인 항목 기준을 문서에 반영했다.
- 운영 확인 항목 표시는 별도 체크리스트가 아니라 버튼형 액션 완료 표시로 처리하기로 확정했다.

## 10. 단계별 상태 전이 조건 v1
### 1) 예약중 → 오늘배차
- **배차일이 오늘인 예약**은 오늘배차 탭 대상으로 본다.
- 배차준비완료 여부와 무관하게 오늘 출차건이면 오늘배차에 노출한다.
- 단, 준비가 아직 안 된 건은 리스트에서 **주황색 표시**로 구분한다.

### 2) 오늘배차 → 배차중
- **탁송 출발**이 배차중 전이 기준이다.
- 오늘배차 탭의 개별 액션들은 독립 액션이지만,
  실제 탁송이 출발한 시점부터 배차중으로 본다.

### 3) 배차중 → 반납일
- **반납일 당일 00:00**부터 반납일 탭으로 넘긴다.
- 반납 하루 전에는 아직 배차중 탭에 두되, **노란색 표시**로 임박 상태를 표시한다.

### 4) 반납일 → 완료
- **반납완료** 액션이 완료 기준이다.
- 실제 반납 종료 처리 후 완료 탭으로 이동한다.

### 5) 완료 → 탭 제외
- 완료 탭은 반납완료 후 **7일간** 노출한다.
- 7일이 지나면 완료 탭에서는 제외한다.
- 이후에는 검색으로만 조회한다.
