# rentcar00_OPS 탭 중심 설계 준비안 v1

## 1. 목적
다음 단계의 기능 설계는 페이지 일반론이 아니라,
**예약 상태별 탭 리스트 중심**으로 다시 작성한다.

이 문서는 그 재작성 준비안이다.

---

## 2. 고정 기준
- 앱 메인은 예약 상태별 탭 리스트다.
- 상세는 보조다.
- 원본은 Google Sheets `예약` + `일정` 탭이다.
- 업무 상태/로그는 Supabase가 관리한다.
- 최종적으로 시트에는 필요한 상태 체크만 반영한다.

---

## 3. 설계 1순위 대상
### 3-1. 예약중 탭
정할 것:
- 이 탭에 들어오는 조건
- 카드에 보여줄 핵심 정보
- 여기서 바로 눌러야 하는 버튼
- 다음 상태로 어떻게 넘기는지

### 3-2. 오늘배차 탭
정할 것:
- 오늘배차 판정 기준
- 카드 우선 표시 정보
- 배차 준비/안내/요청 버튼
- 처리 후 상태 이동

### 3-3. 배차중 탭
정할 것:
- 배차중 유지 기준
- 이용중 체크 항목
- 반납 전 안내/연장 확인 버튼
- 처리 후 상태 이동

### 3-4. 반납일 탭
정할 것:
- 반납일 판정 기준
- 반납 관련 우선 정보
- 반납 안내/회수/완료 버튼
- 완료 후 이동 규칙

### 3-5. 완료/기록 탭
정할 것:
- 완료/종료 기록 조회 기준
- 검색 기준
- 다시 확인할 정보 범위

---

## 4. 보조 설계 대상
### 4-1. 상세 화면
- 메인이 아니라 보조
- 카드에서 다 못 담는 확인용 정보
- 추가 로그/메모 확인
- 예외 처리용 액션

### 4-2. 문자/요청문 생성
- 어떤 탭 액션에서 진입하는지
- 템플릿/문구/복사/공유 흐름

---

## 5. 다음 문서에서 반드시 적을 항목
각 탭마다 아래를 반드시 적는다.
- 탭 목적
- 진입 조건
- 카드 표시 필드
- 핵심 버튼
- 버튼별 상태 변경
- 로그 저장
- 시트 상태 체크 반영 여부
- 예외 상황
